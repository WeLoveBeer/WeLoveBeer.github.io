从源码编译Qt5.8

编译参考的这两个blog:

Qt官网链接

CSDN的一个blog

过程：

  • 下载qtsource 到本地

  • visual studio 2015；ActivePerl; python2

  • 打开vs的命令行工具 然后执行：configure -prefix E:\QtSource\build -debug-and-release -opensource -static -force-debug-info -opengl dynamic -platform win32-msvc -c++std c++11 -skip qtmultimedia -skip qtsensors -skip qtwebengine -skip qtgamepad -nomake examples -nomake tests -mp -skip qtlocation -skip qtserialbus -confirm-license

参数说明：

-prefix 生成目标库的目录

-release 生成release版本  -debug 生成debug版本  -debug-and-release生成debug和release版本

-opensource 编译开源版  -commercial 编译商业版 （参数可选）

-static 生成静态库lib文件  -shared 生成动态库dll文件 （参数可选，默认是shared）/*** 注：静态库与动态库区别是在生成目标文件的链接方式上，静态库采用静态链接，在生成的目标文件（如.exe文件）中会包含库文件代码，而动态库在程序编译时并不会被连接到目标代码中，而是在程序运行时才被载入。***/

-force-debug-info 强制生成调试信息 （参数可选）/*** 这个参数是让Qt release版本也生成.pdb文件。这个文件有什么作用呢？它就是我们在release版本下设置了异常捕获信息函数，打印出错堆栈信息所必须的文件。如果程序没用开启异常捕获选项，是不需要开启这个参数的。***/

-opengl dynamic 指定opengl库 (参数可选） /*** 关于这个参数说明请参考doc.qt.io/qt-5/windows-requirements.html ***/
  • nmake 然后 nmake install

问题总结：

  • qtuiohandler.cpp中有一个字符没能识别，出现 <NMAKE:fatal error U1077> 这样的错误；手动把这个字符改了；

知识点：

  • 从源码编译有三个步骤：

    • Configure:用来生成Makefile文件的,可以配置路径什么的；

    • make: 真正的源码编译；

    • make install : 安装

成长之路

自由之理财


钱是如何都绕不过去话题，摘录一些《财富自由之路》里的一些句子。

  • 财富的定义是：当你完全不工作时，在你现有的生活水平之下你能存活的天数。

  • 你一直在努力工作并使别人变得富有吗？（给别人打工，自己得到的永远是最少的那部分）

  • 你的老板不能使你富有。

  • 你拥有的第一项负债，都是别人的资产。

  • 良性的债务是别人替你支付的债务，不良的债务是你有自己的血汗钱支付的债务。

  • 穷爸爸：成为一名高薪雇员；富爸爸：上学，毕业，创办企业，做一名成功的投资者。

  • 富爸爸认为下面这些才是最重要的：

    • 有时间教育孩子
    • 有时间和金钱照顾自己的健康
    • 能够和家人周游世界
    • 创造就业

  • 大部分富人70%的收入都来自投资，只有不到30%的收入来自工资。

  • 你怎么看我不重要，重要的是我怎么看我自己。

  • 学会如何面对拒绝，如何不受他人意见的影响，以及如何领导他人，你就有望成功。领导力是完全可以学的。

  • “你不能这样做”并不一定意味着“你不能”，大多数情况下是“他们不能”。

  • 投资没有风险，没有文化才有风险。

  • 如果你有太多的债务，世界就会拿走你所有的东西–你的时间，你的工作，你的家庭，你的生活，你的信心，然后再拿走你的尊严。

自己在财务上的最终目标大概如此，罗永浩说：绝大多数取得不凡成就的人，都始于一个不甘于平凡的朴素意愿。加油自己！

平庸的一天

早上闹钟没响！！！

今天早上第一次没听见闹钟响，或者是闹钟压根就没响，没有正常而规律的早起让我十分慌张，打了个车去公司，不好没迟到。生活不会告诉你该怎么办，只会推着你向前走。

工作即是工作，能带来很少的成就感，说实话现在感到十分地迷茫，总感觉在等一个人，在等一件事，在等某一个时刻，仿佛只要这个时刻到来，自己就能华丽地蜕变。但是自己想要什么都不知道，又怎么能期望蜕变出来新的自己会是自己满意的呢?

心理学说幸福感的获得有三个因素：

满足归属需要，并与别处两种人类需要（自主感和能力感）保持平衡，通常就会带来深深的幸福感。

看来现在三样每一样都不如人意，没有归属感：前段时间有个同学说要在杭州定居了，问我之后的打算，我很犹豫，在杭州确实没归属感，没有自己的房子，没有知心的朋友，没有能让自己内心不满与无助说出来的时刻。唯一能带来快感的恐怕只有篮球了；自主感就是自由吧，上班不自由就是最大的不自由；能力感上，虽然现在能胜任当前的工作，但是感觉现在进步的很慢，没有考研时候的那种积极向上的动力，只有盼着假期的心。

投资未来的人是忠于现实的人

自己也在想怎么投资未来，自己的未来是什么样子的，现在想想，自己现在确实没有能拿出手的能力，工作能力一般般，社交能力，情商一般般，智商一般般，口才一般般，奔三的人了，脸皮还如纸一样薄。至少现在已经开始思考未来，开始理财。。。

其实人成长的过程也是在不断实现更大程度的自由，能力越强，可选择的路就越多，，反过来，有越多选项的人，能力也就越强，越能感到自由；就像自己从事技术工作的，能力覆盖的越广，未来越坦荡，只是现在自己的选项很少。

可能是自己想的太多吧，或者是要求的太多，或者是自己不懂感恩，总觉得达不到预期。
天下大事必作于细，天下难事必作于易。时间总还是有的，时间对于每个人来说都是公平的，决定未来的就是现在。

And not, when I have to die, discover that I had not lived!

多态

参数类型和个数不能变，不然使用基类指针将不能调用派生类的函数，这个函数将只能被派生类对象或指针调用。

yield关键字

生成器不用把所有东西装入内存，而是运行一次计算一次，generator相当于里面存了个函数。
带有yield 的函数支持迭代器协议，每次yield返回一个生成器，然后下次调用的时候接着往下执行。

用来实现协程。
最大的优势就是协程极高的执行效率。因为子程序切换不是线程切换，而是由程序自身控制，因此，没有线程切换的开销，和多线程比，线程数量越多，协程的性能优势就越明显。

第二大优势就是不需要多线程的锁机制，因为只有一个线程，也不存在同时写变量冲突，在协程中控制共享资源不加锁，只需要判断状态就好了，所以执行效率比多线程高很多。
